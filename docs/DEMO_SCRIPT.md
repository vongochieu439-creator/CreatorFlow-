# Demo 视频讲解稿

视频建议控制在 3 到 5 分钟，使用声音完整讲解，覆盖核心模块。

## 1. 开场

大家好，这是 CreatorFlow 多平台内容智能发布助手。它解决的是创作者在公众号、知乎、B站、小红书同步发布内容时，需要反复改标题、改结构、补标签、调语气的问题。

页面采用流程式工作台设计。创作台第一屏只让用户输入原始内容，提交后再进入四个平台稿工作台；左侧还可以切换 AI策略和发布交付。

## 2. 输入主稿

这里是原始内容输入区。我可以输入一篇通用稿件，也可以点击“载入样例”。除了标题和正文，还可以设置目标受众、改写语气、是否自动生成标签、是否添加互动引导。点击“生成四个平台稿”后，页面会进入四个平台稿工作台。

下面的“参考样本”区域可以粘贴用户自己提供的优质内容样本。系统不会复制样本文案，而是让 AI 分析它们的标题规律、开头方式、段落节奏、标签和互动方式。

如果想更明确地分析各平台爆款策略，可以进入左侧“AI策略”。这里支持两种方式：第一种是粘贴公开链接，点击“采集并分析链接”，后端会提取公开页面的标题、摘要、结构线索和正文片段，再交给真实 AI 分析；第二种是直接把用户已授权的爆款样本粘贴到四个平台样本框，再点击“分析平台样本”。页面会标注“真实样本分析”和样本依据，不会把预置策略伪装成真实结果。

## 3. 多平台适配

生成后会看到四个平台稿工作台，可以切换公众号、知乎、B站和小红书。正式优化时，我可以点击“一键智能优化”，系统会通过真实 API 生成更贴近平台风格的版本。如果只是不满意某一个平台，比如小红书，也可以点击“优化当前平台”，只重写当前平台稿，不影响其他三个平台。

这一步的产品价值是：工具不是简单改写，而是先理解平台内容规律，再基于用户主稿生成平台版本。

如果本地配置了大模型 API，智能优化会通过本地 `server.js` 代理请求模型。前端不会直接保存 API Key；调用失败时，系统会保留当前稿，避免覆盖用户已经满意的版本。

如果提供了参考样本或在 AI策略页分析了平台样本，真实 AI 会把样本作为分析材料，页面会显示策略来源为“用户参考样本 + 真实 API”。这比直接爬取平台内容更合规，也更适合用户把自己的行业样本带进来。

工具内置了四个平台：

- 公众号：更适合完整长文，所以会生成背景、核心拆解和总结式表达。
- 知乎：更强调结论先行和原因分析。
- B站：会生成视频简介和时间轴。
- 小红书：更偏短句、清单和标签分发。

切换平台卡片，可以看到标题、摘要、正文结构和标签都会变化。策略卡片也会展示当前平台为什么这样生成。

## 4. 发布前检查

右侧检查区会展示标题长度、正文字数、标签和平台风格检查。内容指标区会统计字数、标题长度、标签数和结构块，帮助用户在发布前快速判断是否符合平台习惯。

内容评分现在接入真实 AI 质检接口，并由用户主动触发。系统会把当前平台稿、平台画像、目标受众和策略一起发送给本地 `server.js` 代理，由大模型给出综合分与可行动建议。评分结果会显示在当前稿质检区，也会绑定到对应保存版本卡片；如果 API 没配置，页面会明确提示质检未完成，不会用模拟分冒充真实结果。

在创作台的当前平台稿下方，还有“AI改写说明”。这里会解释当前平台为什么采用这套策略、AI具体改了哪些地方、参考样本影响了什么、以及发布前要注意的风险。这样可以降低 AI 改写的黑盒感，也方便用户判断结果是否可信。

## 5. 模拟发布

AI 生成后，用户可以直接在左侧平台稿上修改标题、摘要和正文。保存当前版本后，系统会把它记录到版本历史里，用户不满意也可以回退到历史版本或恢复 AI 版本。这体现了产品设计里的一个原则：AI 负责辅助生成，最终发布前仍由用户确认。

创作台右侧的“版本审查”不会在没有人工修改时展示无意义的相同对比。用户保存人工版本后，它会对比 AI 原稿和当前稿，并总结改了哪些字段、正文增减了多少字、是否建议重新评分。

如果暂时不接真实平台接口，可以点击“导出全部稿”，系统会生成一份 Markdown 发布包，包含四个平台的最终稿、策略重点和风险提醒，用户可以拿去各平台后台发布。

在发布交付页，用户可以为每个平台选择发布版本：使用 AI 当前稿，或者选择刚才保存过的人工版本。导出和模拟发布都会读取这里的选择，这样版本管理不只是审查，而是真正进入发布决策。

点击“模拟一键发布”，系统会通过 MockPublisher 把当前内容生成到四个平台，并在发布总览和发布记录中显示“发布中”和“已模拟发布”等状态。真实上线时，这一层可以替换为各平台开放接口或自动化发布服务。

右侧“发布通道”说明了当前版本为什么采用 MockPublisher，以及未来每个平台的接入方向：公众号可以接草稿箱，B站可以接投稿流程，小红书和知乎可以先做半自动发布或分享式发布。这样既保证 Demo 稳定，也保留了真实产品扩展路径。

## 6. 架构说明

项目的核心是平台适配器。每个平台都有自己的标题限制、理想长度、内容结构和 adapt 方法。未来要扩展新平台，只需要新增一个 Adapter，不需要重写整个应用。

AI 能力方面，当前版本已经通过本地代理接入 OpenAI-compatible 接口。模型负责标题、摘要、正文风格迁移、标签生成和真实质检评分；规则层只负责基础格式、指标展示和发布流程编排。

## 7. 收尾

这就是 CreatorFlow 的完整流程：输入一份主稿，自动生成多平台发布稿，完成预览、检查和模拟发布。谢谢观看。
