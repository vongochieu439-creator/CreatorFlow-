# 提交检查清单

## 必交内容

- 公开 GitHub 或 Gitee 仓库，截止后仍可公开访问。
- README 文档。
- Demo 视频外链，链接放在 README 显眼位置。

## 有效性检查

- 作品符合“多平台内容发布工具”选题方向。
- 代码仓库在题目发布后创建。
- 开发周期内持续提交 commit 和 PR，避免最后一天一次性提交所有代码。
- 所有 commit 时间都在本批次开始和截止时间内：5月29日 00:00 到 5月31日 23:59。
- PR 描述不能空，且必须和实际代码变更一致。
- 如果引用第三方库或框架，需要在 README 中明确依赖，并说明原创功能部分。
- 如果复用过往代码，需要在 PR 描述中注明来源。

## PR 规范

- 每个 PR 只做一件事。
- PR 标题一句话说明新增或修改了什么。
- PR 描述包含功能描述、实现思路、测试方式。
- PR 合并后，主分支代码保持可运行。

## Demo 视频

- 使用声音完整讲解。
- 展示核心功能和主要效果。
- 视频上传至可访问的外部平台。
- README 中放置视频链接，不要藏在子目录或文件夹中。
