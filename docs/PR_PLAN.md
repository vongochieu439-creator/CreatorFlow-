# PR 拆分计划

根据比赛提交要求，开发期间应持续提交 PR 和 commit，避免最后一天一次性导入全部代码。

## PR 1：初始化 CreatorFlow 项目结构

功能描述：

- 新增静态 Web 项目结构。
- 完成基础页面布局。
- 增加内容输入区和平台预览区骨架。

实现思路：

- 使用纯 HTML、CSS、JavaScript，降低运行门槛。
- 首屏直接进入可用工具界面，不做营销落地页。

测试方式：

- 打开 `index.html`，确认页面可加载。
- 调整浏览器宽度，确认布局不明显错位。

## PR 2：实现平台适配器与多平台预览

功能描述：

- 新增公众号、知乎、B站、小红书四个平台适配器。
- 根据主稿生成标题、摘要、正文结构和标签。
- 支持平台卡片切换预览。

实现思路：

- 在 `src/platforms.js` 中定义 Adapter。
- 每个平台维护自己的标题限制、理想长度、内容结构和 adapt 方法。

测试方式：

- 修改主稿后，确认四个平台预览实时变化。
- 切换平台，确认不同平台输出风格不同。

## PR 3：增加发布前检查、指标与模拟发布

功能描述：

- 增加标题长度、正文字数、标签、平台风格检查。
- 增加内容指标统计。
- 增加模拟一键发布和发布记录。
- 使用 localStorage 保存编辑内容和发布记录。

实现思路：

- 由 Adapter 输出 checks 和 metrics。
- 发布层统一读取所有平台草稿并生成本地记录。

测试方式：

- 点击“模拟一键发布”，确认生成四个平台发布记录。
- 刷新页面，确认记录保留。

## PR 4：补充 README、Demo 脚本和架构说明

功能描述：

- 完善 README。
- 增加 Demo 视频讲解稿。
- 增加三天开发计划和 PR 拆分计划。

实现思路：

- README 对齐提交要求，说明选题方向、核心功能、架构设计、AI 能力设计和测试方式。

测试方式：

- 按 README 步骤打开项目并完成一次端到端演示。
